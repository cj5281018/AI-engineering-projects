一、背景
智能客服的知识库经过一年多的积累，有 500 多条 FAQ。但没人维护过，可能存在内容过时、不同条目之间说法矛盾、有些问题根本没覆盖到等问题。
业务方希望做一次知识库"体检"，找出有问题的条目并给出改进建议。

二、任务要求
1. 定义"问题条目"类型
从附件中的知识库数据出发，自己判断哪些条目属于"有问题"的。可能有多种问题类型，你需要定义分类并说明每类的影响。
2. 实现检测工具
自动扫描附件中的知识库，标记有问题的条目并说明具体问题。
可用真实 LLM API，也可用 mock 模式模拟返回。
3. 给出治理建议
对有问题的条目，给出具体的改进建议（修改、合并、删除、新增等）。
4. 输出治理报告
整理一份可交给业务方看的报告：多少条目有问题、各类问题分布、优先处理建议。
5. README
简要说明问题分类体系、检测方法、治理建议逻辑、AI 工具使用情况。

三、交付物
请提供以下内容（README 文档 + 截图，或 GitHub 仓库链接）：
1. README：问题分类定义、检测方法、治理报告摘要、AI 工具使用情况
2. 开发工具截图：展示你的开发过程（IDE / Agent 工具 / 终端等）
3. 运行结果截图：治理报告、问题条目列表等
4. 线上地址或 GitHub 链接（推荐）：如果部署到线上或提交到 GitHub，提供链接即可，无需额外提交代码

四、附件
kb_articles.json：40 条知识库 FAQ（故意包含多种质量问题）
business_context.md：当前业务规则摘要（用于判断条目是否过时）
