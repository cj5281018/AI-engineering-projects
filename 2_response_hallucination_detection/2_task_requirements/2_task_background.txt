一、背景
团队的智能客服系统偶尔会"说瞎话"——回复用户时编造不存在的优惠政策、给出错误的退货地址、或者杜撰产品参数。业务方需要一个工具来批量检测这些"幻觉"回复。
二、任务要求
1. 定义"幻觉"分类：附件中的回复数据包含多种类型的幻觉，你需要自己定义分类体系：哪些算幻觉？分几类？每类的严重程度如何？
2. 实现检测工具：对你定义的幻觉类型，实现自动化检测（方法不限）。对附件中 20 条回复逐条检测，标注结果。可用真实 LLM API，也可用 mock 模式模拟返回。
3. 验证检出率：用 ground_truth.json 中的人工标注结果，计算你的检测工具漏检了哪些、误报了哪些。
4. 分析误判原因：哪些 case 容易误判？为什么？
5. README：简要说明分类体系、检测方法、检出率数据、AI 工具使用情况。
三、交付物
请提供以下内容（README 文档 + 截图，或 GitHub 仓库链接）：
1. README：幻觉分类定义、检测方法、检出率、AI 工具使用情况
2. 开发工具截图：展示你的开发过程（IDE / Agent 工具 / 终端等）
3. 运行结果截图：检测结果、检出率数据等
4. 线上地址或 GitHub 链接（推荐）：如果部署到线上或提交到 GitHub，提供链接即可，无需额外提交代码

四、附件
replies.json：20 条客服回复（含用户问题 + 系统回复 + 相关知识库条目）
ground_truth.json：人工标注的幻觉检测结果
