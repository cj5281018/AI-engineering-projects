# AI Engineering Projects

> 将模糊的业务需求落地为可运行的自动化方案。

本仓库包含四个独立的 AI 工程项目，覆盖客服系统的质量评估、异常检测、趋势分析与知识库治理。每个项目从原始数据出发，经过方案设计、代码实现、结果验证到报告输出，形成完整交付链路。

---

## 项目一览

### 1. 客服自动回复质量评估

**需求问题**：团队上线了"客服自动回复"功能，业务方要求评估回复是否"准确、有用、语气好、不瞎编"，但标准模糊，需要用量化指标说清楚。

**交付结果**：定义 4 个可自动评估的指标（Faithfulness、Answer Relevance、Helpfulness、Tone & Empathy），对 20 条测试 case 逐条评分，输出评估报告。

**解决方案**：借鉴 RAGAS 和 G-Eval 学术框架，构建双模式评估流水线（Mock 规则映射 + 真实 LLM API），对每条回复从 4 个维度打分并加权综合。

**结果展示**：
- 综合均分 **4.04 / 5.0**（良好），最低分 3.07
- 最大短板：**Helpfulness（有用性）** 均分仅 3.7
- 最差 3 条 case 深度分析（case_03 退款到账 / case_06 优惠券 / case_20 退货流程）

### 2. 客服回复幻觉检测

**需求问题**：智能客服偶尔会"说瞎话"——编造不存在的优惠政策、错误退货地址、杜撰产品参数，需要批量检测这些"幻觉"回复。

**交付结果**：定义 6 类幻觉分类体系，对 20 条回复逐条检测，与人工标注对比验证检出率。

**解决方案**：构建 Rule + LLM 混合检测架构。3 条确定性规则（KB-Empty / Numeric-Conflict / Keyword-Negation）处理可枚举场景，LLM（DeepSeek / Mock）处理语义推理，最后由 Confidence-Based Fusion 引擎综合判定。

**结果展示**：
- **准确率 100%**（精确率 100% / 召回率 100% / F1 100%）
- 规则主导 5 条，LLM 主导 6 条，混合决策 9 条
- 识别已知难点：半对半错、语义否定推理、信息遗漏边界

### 3. 客服工单趋势分析

**需求问题**：客服主管每天需关注大量工单，但靠人工翻看效率极低，需要自动从工单数据中发现趋势和异常信号。

**交付结果**：基于 50 条工单数据的多维度分析 Dashboard，支持交互式探索和异常自动预警。

**解决方案**：用 Streamlit 构建交互式 Dashboard，覆盖 7 个分析维度（总览 / 时间趋势 / 分类分析 / 处理效率 / 满意度 / 异常检测 / 渠道分析）。异常检测采用三级递进引擎（Z-score 2σ → 7日滚动平均偏离 → IQR 箱线图）+ 业务规则补充。

**结果展示**：
- 支付问题（32%）+ 退款退货（26%）占比 58%，符合帕累托原则
- 整体满意度均值 **2.36/5**，时效性与满意度呈显著负相关
- 自动检测到 **2 次满意度连续下降异常** + 退款退货未解决率异常（38.5%）

### 4. 知识库质量检测

**需求问题**：智能客服知识库有 500+ 条 FAQ，但从未系统维护，可能存在过时、矛盾、覆盖缺失等问题，需要做一次全面"体检"。

**交付结果**：定义 8 类问题条目类型，对 40 条样本数据（含故意设置的质量问题）进行检测，输出带优先级排序的治理报告。

**解决方案**：参考 DAMA-DMBOK、ISO 8000、KCS 等国际标准，设计规则引擎（过时检测 / 空答案 / 重复 / 矛盾）+ LLM 语义检测（专业度 / 语义矛盾 / 建议生成）双轨架构，按 P0→P1→P2 三级优先级输出治理建议。

**结果展示**：
- **健康率 62.5%**，CQS 综合质量得分 **3.8 / 5.0**
- 发现 **15 条问题条目**（P0 立即处理 3 条 / P1 尽快处理 11 条 / P2 常规优化 1 条）
- 最大问题：**时效性**（7 条过时条目，评分仅 3/5）
- 退货政策分类健康率最低（44.4%）
